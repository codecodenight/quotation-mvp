## V4.1 Issues Found

### 1. Health warnings do not recognize `size_display`

`LS-R02A-30W` has a structured size parameter:

```text
Size: 467×244×155mm
```

But preview still warns `缺 Size` because quote health checks only look at `products.size`, not `product_params.size_display`.

Suggested fix: update quote health to treat `size_display`, or length/width/height params, as satisfying the size requirement.

### 2. CCT extraction includes tolerance values as standalone CCT

`XYJ-SWL-1000LM` exported:

```text
CCT: 500K
```

The source likely contains `6500±500K`; `500K` is tolerance, not a valid customer-facing CCT.

Suggested fix: reject standalone CCT values below a practical threshold such as 1800K, unless explicitly configured for a special category.

### 3. Work light details fall back to mixed raw text

`LS-W12F-20W单头` exported raw/fallback details:

```text
PF: 0.9
Voltage: /
Power: 20W±10%
Lumen: 1600LM±10%
产品单灯尺寸(MM): 128*93*28
外箱尺寸(MM) 参考用: 620*280*280
```

This is not clean customer English. It also leaks carton/reference dimensions into Product Details.

Suggested fix:
- Enhance 工作灯 extractor to capture lumens, size_display, PF, and voltage cleanly.
- Filter packaging-only labels such as `外箱尺寸` from Product Details fallback.
- Ignore empty label values like `Voltage: /`.

### 4. New KCD solar wall lights have no useful specs

`TK-13 A Solar wall light` exported only:

```text
Specifications: Chip Type: SMD2835
```

This is a source-data limitation, but it makes the quote weak.

Suggested action: put these products into a "待补规格" queue, or add a data-quality filter for products with fewer than two useful Product Details lines.

### 5. CTN coverage remains the main quote warning source

4 of 5 rows had missing CTN Qty and L/W/H. Export still works, but every row is warned before export.

Suggested action: V4.1 should improve how CTN warnings are grouped and prioritized, so missing CTN does not bury more important problems like malformed Product Details.

## Recommended V4.1 Scope

1. Make quote health checks param-aware: `size_display` and dimension params should satisfy size.
2. Tighten parameter extraction/display rules:
   - Drop invalid CCT values like `500K`.
   - Drop empty values such as `/`.
   - Exclude packaging labels from Product Details fallback.
3. Add a Product Details quality warning:
   - fewer than 2 useful lines
   - contains Chinese labels in customer mode
   - contains packaging-only labels
4. Improve quote preview warnings:
   - separate data completeness warnings from customer-facing text quality warnings.
   - allow "only show customer-facing problems".
