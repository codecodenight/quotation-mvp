# Codex Task: V2.13A — 源文件盘点（9 个优先目录只读扫描）

## 目标

对 9 个数据缺口最大的工厂报价目录做只读扫描，自动分类每个 Excel 文件为 `likely-importable` / `needs-review` / `likely-skip`，产出结构化报告供人工决策。

**不导入、不改 DB、不改源文件。**

## 背景

当前 DB 有 2,144 产品 / 2,235 offers，来自 89 个已导入文件。磁盘上 `各家工厂最新报价汇总/` 下有 ~1,614 个 Excel 文件，大部分未导入。以下 9 个目录的品类缺口最大（缺图、缺 CTN、缺价格、产品数不足）：

1. `室内照明/LED橱柜灯` — DB: 134 产品，CTN 为 0
2. `户外照明 工业照明/市电壁灯` — DB: 27 产品
3. `光源/球泡灯管` — DB: 152 产品，仅 25 有图
4. `光源/灯丝灯` — DB: 471 产品，L/W/H 箱规为 0
5. `室内照明/轨道灯` — DB: 155 产品，CTN 基本为 0
6. `灯带` — DB: 21 产品，磁盘 292 文件
7. `户外照明 工业照明/三防灯` — DB: 79 产品
8. `室内照明/镜前灯` — DB: 63 产品
9. `户外照明 工业照明/LED 地埋灯地插灯` — DB: 58 产品

根目录：`/Volumes/My Passport/AI 报价/各家工厂最新报价汇总/`

## 扫描逻辑

### 对每个 Excel 文件（.xlsx / .xls）：

1. **基础信息**：文件名、完整路径、大小（bytes）、修改时间、所在工厂子目录名
2. **是否已导入**：组合判断，不能只靠路径匹配
   - 匹配 `files` 表的 `file_name`（完全匹配或去掉日期后缀后匹配）
   - 匹配 `file_size` + `modified_at`
   - 检查该 file 是否有关联的 `supplier_offers`（`source_file_id`）
   - 三项中有两项匹配 → `已导入`
   - 只有文件名类似 → `可能已导入（需确认）`
   - 都不匹配 → `未导入`
3. **Sheet 结构预览**：用 SheetJS 读每个 sheet
   - sheet 名
   - 总行数
   - 前 10 行采样：识别疑似表头行（含"型号" / "Model" / "单价" / "Price" / "价格" / "报价" 等关键词的行）
   - 疑似价格列：至少 5 行含正数的列（排除年份 >2000 的整数列）
   - 疑似型号列：至少 5 行含字母+数字混合值的列
4. **分类打标**：
   - `likely-importable`：有表头行 + 有疑似价格列 + 有疑似型号列 + 未导入
   - `likely-skip`：已导入 / 无价格列 / 空文件 / 纯图片目录 / 文件名含"目录" "画册" "说明书" "知识" "样册"
   - `needs-review`：不符合上面两类的其他文件（有价格但没型号，或结构不确定）
5. **多版本检测**：同一工厂子目录下文件名相似（去掉日期后缀后相同）的文件归为一组，标出最新版本

### 不做的事：

- 不解析具体的价格值或产品数据
- 不做 mapping 配置
- 不写入 DB
- 不修改源文件
- 不处理 PDF 文件（只统计数量）

## 执行步骤

### Step 1: 检查外接硬盘

- 确认 `/Volumes/My Passport/AI 报价/各家工厂最新报价汇总/` 可访问
- 如果不可访问 → 报告并停止
- 列出 9 个目录的文件数

### Step 2: 加载已导入文件清单

- 从 DB 读取所有 `files` 表记录（file_name, file_size, modified_at, id）
- 读取哪些 file.id 有关联 supplier_offers
- 构建查找表供后续匹配

### Step 3: 逐目录扫描

- 递归扫描每个目录下所有 .xlsx / .xls 文件
- 对每个文件执行上述扫描逻辑
- 跳过 >100MB 的文件（标为 `needs-review`，记录大小）
- 对 .xls 文件：SheetJS 同样可读，正常处理

### Step 4: 生成报告

- 按上述输出格式写入 `docs/v2.13a-source-inventory.md`
- 报告末尾附上扫描统计：总文件数、总耗时、跳过文件数、读取失败文件数

### Step 5: 提交

- git commit

## 注意事项

- 源 Excel 文件绝不修改
- DB 不写入任何数据
- 外接硬盘如果未挂载，Step 1 直接停止，写报告说明
- SheetJS 读文件如果报错，记录错误但继续扫描下一个
- 扫描脚本写在 `scripts/source-inventory.ts`，可复用于后续全量扫描
