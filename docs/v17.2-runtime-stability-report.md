# V17.2 Runtime Stability Report

Date: 2026-06-21

## Summary

V17.2 stabilizes local runtime startup for the quotation MVP on a 16GB Mac.

Changes made:
- Added a 2GB Node heap cap to the dev script.
- Made dev startup clear stale `.next/dev` lock state before starting.
- Made dev/build explicitly use Turbopack.
- Added a production helper script.
- Added Watchman ignore rules for large local data/output folders.

## Code Changes

### package.json

- `dev`: `rm -rf .next/dev && NODE_OPTIONS='--max-old-space-size=2048' next dev --turbopack`
- `build`: `next build --turbopack`
- `prod`: `npm run build && next start`

### next.config.ts

- Kept `allowedDevOrigins: ["127.0.0.1"]`
- Added explicit `turbopack: {}`
- Kept `outputFileTracingExcludes` for local data/output/sample directories

### .watchmanconfig

Ignored large local-only directories:
- `data`
- `backups`
- `outputs`
- `sample-data`
- `sample data`
- `node_modules`

## Watch Exclusion Decision

The task asked to exclude `data/`, `backups/`, `outputs/`, and sample folders from file watching.

Findings:
- Next 16.2.7 exposes no supported Turbopack watch-ignore option for these directories.
- A webpack `watchOptions.ignored` override was tested, but it caused production `/quotes` to fail with a Next manifest initialization error.
- A webpack production build path was also tested and did not complete in a reasonable time on this project.

Decision:
- Use Turbopack explicitly for dev/build.
- Use `.watchmanconfig` to keep local file watching away from large data/archive directories.
- Do not keep the webpack override because it made the runtime less stable.
