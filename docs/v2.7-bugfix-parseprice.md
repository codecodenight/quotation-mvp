# V2.7 Bugfix — parsePriceValue RMB Symbol Priority

Generated: 2026-06-09

## Problem

`parsePriceValue()` previously extracted the first number in a cell.

This was wrong for #25 where price cells include product/spec numbers before RMB prices, for example:

```text
15000MA ¥282.5
10000MA ¥240.0 15000MA ¥246.0
```

The old parser would import `15000` or `10000` as `supplier_offers.purchase_price`.

## Fix

If cell text contains `¥` or `￥`, `parsePriceValue()` now extracts the first number after the first RMB symbol.

Cells without an RMB symbol keep the previous behavior: extract the first valid positive number from the cell text.
