## 使用说明

每个品类-工厂组合有一个 **决定** 列：
- `✅` = 导入（默认建议）
- `❌` = 跳过
- `⏸` = 暂缓（等后续批次）
- `❓` = 需讨论

修改后保存，作为 V2.14 批量导入的输入。

### Codex 审阅标注原则

- V2.14 Batch 1 只做数据缺口最大且品类语义清楚的 5 个品类：投光灯 / 面板灯 / 线条灯 / 路灯 / 灯带。
- 其他现有品类先标 `⏸`，进入 Batch 2/3，避免一次导入 683 个文件导致质量问题难定位。
- 新品类中，风扇灯 / 工作灯 / G4G9 值得新建，但不进 Batch 1；配件类（铝型材 / 灯带连接器）先不进产品库。
- 混合品类或需要按文件内容拆分的行标 `❓`，不直接进批量导入。

## 五、导入策略建议

### 多文件工厂问题

一些工厂有大量文件（如一群狼 44 面板灯、凯晟德 27 投光灯、普照 35 三防灯）。这些多为：
- **版本迭代**：同产品的不同日期报价 → 只导最新版
- **不同产品线**：同工厂不同系列 → 都要导

**建议策略**：每个工厂全部导入，依靠 V2.10 的 upsert 逻辑（product_id + factory_name）自动去重——同型号更新价格，新型号新增产品。

### 预计规模

| 场景 | 预估文件数 | 预估新产品 |
|---|---:|---:|
| 全部导入（683 文件） | 683 | ~21,000 |
| 去掉新品类 + 未判定 | ~618 | ~18,700 |
| 仅高价值品类 | ~524 | ~16,800 |

### 分批建议

一次导入 683 个文件风险高。建议分批：

1. **Batch 1**：数据缺口最大的品类（投光灯/面板灯/线条灯/路灯/灯带）— ~309 文件
2. **Batch 2**：有一定数据但需补充的品类（吸顶灯/筒灯/三防灯/磁吸灯/净化灯/镜前灯/防潮灯）— ~230 文件
3. **Batch 3**：全新品类 + 低优先级 — ~144 文件
