## Step 2 — `parseMultiPrice()`

Added `parseMultiPrice(value)` in `src/lib/excel-import.ts`.

Rules implemented:

- Parses 2+ `variant:price` pairs.
- Supports spaces, commas, Chinese commas, semicolons, and Chinese semicolons between pairs.
- Supports `:` and `：`.
- Rejects single-pair values such as `3CCT:9`.
- Rejects normal single-price values such as `¥9.5`.
- Does not change `parsePriceValue()`.

Important implementation note: `buildHejiaImportRows()` checks `parseMultiPrice()` before `parsePriceValue()`. This is necessary because the existing `parsePriceValue()` intentionally remains unchanged and would otherwise read `3CCT:9 12CCT:10.5` as `3`.

## Step 3 — Hejia Import Split

Updated `buildHejiaImportRows()` so one source row can create multiple product/offer pairs when the price cell contains multi-price entries.

For each variant:

- shared fields are preserved: factory, currency, MOQ, CTN Qty, CTN L/W/H, size, description
