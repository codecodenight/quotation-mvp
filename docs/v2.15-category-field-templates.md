# V2.15 — 品类字段模板定义

> 状态：**已确认 — V3.0 核心输入文档**
> 生成日期：2026-06-10
> 数据来源：当前 DB 中 2,144 产品 / 2,235 offers 的实际样本

---

## 目标

为每个品类定义一组 **结构化参数字段**，作为 V3.0 参数提取的目标模板。

**不是替代 `product_name`，是补充结构化参数。** 现有字段（product_name / model_no / remark / size / material）全部保留不动，V3.0 从这些字段以及源 Excel raw row 中提取参数，写入新表 `product_params`。

### 当前问题

| 品类 | 现状 | 示例 |
|---|---|---|
| 球泡 | 参数散落在 product_name / remark / size 里 | Watts 和 Base 混在文本里，不可筛选 |
| 太阳能 | product_name 含 100+ 字规格长文本 | Panel/Battery/Lumens/CCT 全堆在一起 |
| 灯丝灯 | product_name = 中英文混合名 | 无 Watts/CCT/Lumens 可查询字段 |
| 轨道灯 | model_no 隐含功率数字 | K1220-8A 的 `8` 不一定是 8W |

### 目标状态

- 每个品类有预定义的参数字段列表
- V3.0 的提取器根据字段模板从 `model_no / remark / size / material` 及源 Excel raw row 中提取参数
- 提取结果存入新表 `product_params`
- 导出报价时 Product Details 列继续合并显示，但从结构化字段生成（更准确、可排序）
- 现有报价链路不受影响

## 数据模型（已定）

### key-value 表 + raw/normalized 双值

```
product_params
  id                TEXT PK
  product_id        TEXT FK → products
  param_key         TEXT        -- e.g. "watts", "base", "cct"
  raw_value         TEXT        -- 原始文本，如 "Φ50*55"
  normalized_value  TEXT        -- 标准化值，如 "50" (对应 diameter_mm)
  unit              TEXT        -- e.g. "W", "mm", "K"
  source_field      TEXT        -- 提取来源：model_no / remark / size / material / raw_row
  confidence        TEXT        -- high / medium / low
  created_at        DATETIME    DEFAULT CURRENT_TIMESTAMP
  updated_at        DATETIME
```

**设计原则：**
- `raw_value` 保留原始来源文本，不清洗
- `normalized_value` 是提取后的标准化值，用于筛选和计算
- `confidence` 标记提取可信度：
  - `high`：文本中有明确单位（如 `18W`、`IP65`、`3000K`）或表头明确标注
  - `medium`：格式匹配但无明确单位（如 size 字段中的数字）
  - `low`：需人工确认（如从型号数字推断功率）
- 多值字段用多行存储（同产品同 key 多条记录），显示时合并

## 提取安全规则

### 禁止按品类默认值填参数

- ~~三防灯默认 IP65~~ → 只有文本中明确出现 `IP65` / `IP66` 等才提取
- ~~地埋灯默认 IP67~~ → 同上
- 品类可以定义 `expected_params`（预期参数列表），但不能作为实际 `normalized_value`

### 从 model_no 提取 watts 要谨慎

- 只有带 `W` 后缀（如 `18W`、`36W`）或源文件表头明确标注"功率/Power/Wattage"时，才标 `confidence=high`
- 型号中的纯数字（如 K1220-**8**A）最多标 `confidence=low`，不进入报价展示
- 禁止将系列号、尺寸、孔径等数字当功率

### size 拆分策略

`products.size` 原始值不动。`product_params` 中按需拆出维度字段：

| param_key | 说明 | 示例 raw_value → normalized_value |
|---|---|---|
| diameter_mm | 直径 | `Φ50*55` → `50` |
| length_mm | 长度 | `1200*105*69.5` → `1200` |
| width_mm | 宽度 | `1200*105*69.5` → `105` |
| height_mm | 高度 | `1200*105*69.5` → `69.5` |
| size_display | 客户可读尺寸 | `Φ50*55` → `Φ50×55mm` |

## 字段优先级

V3.0 提取器应按以下优先级处理字段：

### P0 — 必提取（报价刚需）
- `watts` — 几乎所有品类的核心参数（必须有 W 后缀或表头明确）
- `length_mm` / `diameter_mm` — 线性灯具/圆形灯具的核心尺寸
- `base` — 球泡/灯丝灯/灯管必需
- `ip` — 户外/防水品类必需（必须文本中明确出现 `IPxx`）

### P1 — 应提取（客户常问）
- `cct` — 色温（必须有 K 后缀）
- `lumens` — 光通量（必须有 lm 后缀）
- `material` — 材质（已有 products.material，但填充率低）
- `dimmable` — 可调光
- `beam_angle` — 光束角

### P2 — 可选（数据完善）
- `cri` — 显色指数
- `voltage` — 电压
- `certification` — 认证
- `warranty` — 质保
- `sensor` — 感应方式

## 已定决策

1. **product_name 不动。** 不批量简化、不重写。V3.0 只新增结构化参数，不破坏现有报价链路。

2. **导出用方案 A：Product Details 继续合并显示。** 从结构化字段生成更准确的合并文本。不做按品类展开多列，避免客户报价模板不稳定。后续内部模式可选加"展开参数列"。

3. **多值字段用多行 key-value。** 同产品同 key 可有多条记录（如 cct=3000K, cct=4000K, cct=6500K）。显示时合并为 `3000K / 4000K / 6500K`。不用逗号字符串。

4. **size 不改原始值。** `products.size` 保持原样。`product_params` 中新增标准化维度字段（`diameter_mm` / `length_mm` / `width_mm` / `height_mm` / `size_display`）。


### V3.0 提取器的核心约束（从本文档继承）

1. **只新增，不覆盖** — 现有字段全部保留
2. **保留 raw + normalized** — 可追溯提取来源
3. **标注 confidence** — low 不进入报价展示（见下方规则 6）
4. **禁止品类推断** — IP、功率等必须有文本依据
5. **多值用多行** — 不用逗号字符串
6. **提取来源按优先级**：`products.remark / size / material / model_no` → `supplier_offers.remark` → 源 Excel raw row。不是每个 product 都能稳定回到原始 Excel 行，DB 内字段优先。
7. **报价展示只用 high / medium** — `confidence=low` 可以存入 `product_params`，但不进入客户报价单，不自动覆盖 Product Details。仅作为"待人工确认"标记。
