## 关键发现

### 1. 提取效果因品类差异极大

- **高提取率品类**（现有 DB 字段就够）：球泡、太阳能、灯带、净化灯、吸顶灯
- **部分可提取品类**（核心字段缺，需源文件补）：灯丝灯(缺watts/base)、三防灯(缺watts/IP)、筒灯(部分缺watts)
- **低提取率品类**（DB 字段信息不足）：轨道灯、磁吸灯、应急灯、地埋灯

### 2. V3.0 第一阶段可以不等源文件

对球泡、太阳能、灯带、净化灯、吸顶灯这 5 个品类（合计 ~700 产品），现有 DB 字段就能提取出大部分参数。V3.0 可以先对这些品类做 dry-run，验证提取管线。

### 3. 灯丝灯是最大的"半成品"

471 产品（最大品类），shape/glass/size 提取率 100%，但最关键的 watts 和 base 只有 37%。这 63% 的缺失只能靠 V2.14 从源 Excel 补。

### 4. size 标准化是高回报低风险的第一步

所有品类的 size 字段填充率都很高，但格式混乱（Φ、Ø、Dia、cm/mm 混用、L/W/H 前缀可选）。只做 size 拆维度（diameter_mm / length_mm / width_mm / height_mm）就能覆盖绝大多数产品。

### 5. remark 字段是被忽视的金矿

球泡的 remark 含 voltage、cri、dimmable；三防灯的 remark 含 material；太阳能的 product_name 本身就是完整规格书。V3.0 提取器不能只看 model_no。

### 6. 存在误导入数据

球泡样本中有 1 条表头行被当产品导入（model_no = `LED Bulb-Power-Lumen`）。这类数据清理应在 V2.14 之前做。

---

## 结论

**V2.15 模板有实际价值，V3.0 值得做。** 但策略应该是：

1. **V3.0 Phase 1**：先处理"高提取率"品类（球泡、太阳能、灯带、净化灯、吸顶灯，~700 产品），用现有 DB 字段，不需要等源文件
2. **V2.13A → V2.14**：补齐源数据，重点补灯丝灯 watts/base、三防灯 watts/IP、轨道灯 watts
3. **V3.0 Phase 2**：处理"低提取率"品类，依赖源 Excel raw row 作为提取来源
4. **前置清理**：size 标准化 + 误导入数据清理可以先做，不依赖硬盘
